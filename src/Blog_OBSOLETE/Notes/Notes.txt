Build & Deploy Procedure:

Remember to change hosts file as needed.
login to cp.winhost.com after publishing and recycle service
note that blog.api appsettings.* are not checked into source control

Blog Project ====================================================================
MAKE A BACKUP OF PROD SERVER WEB.CONFIG IN WWW AND API PROJECTS 
Open a command window in c:\git\blog\src\blog
In VS, delete content of www root\build folder (it gets populated by buildProd)
run: tsc
run: npm run-script buildProd <- case sensitive
Optionally, delete content of deployment target folder i.e. c:\inetpub\SamWheat or ftp.samwheat.com
Right click on project in VS and click Publish
"logs" directory and "node_modules" directory must exist on prod server even if they are empty which they should be.
"node_modules" dir is at the same level as wwwroot
===================================================================


Blog.API Project ===================================================================
After publishing, ftp to site and copy /api/web.config to local temp dir.
remove handlers section
may need to add environment variables section:
<environmentVariables>
	<environmentVariable name="ASPNETCORE_ENVIRONMENT" value="Prod" />
</environmentVariables>
===================================================================

For debugging API:  change this:
     <environmentVariable name="ASPNETCORE_ENVIRONMENT" value="Prod" />
to this:
     <environmentVariable name="ASPNETCORE_ENVIRONMENT" value="Development" />
also change connection string in appsettings.development.json
